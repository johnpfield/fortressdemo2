Section V - Install Fortress


This section provides instructions to install and configure the Fortress IAM system and configure for SSL connectivity with OpenLDAP. Everything needed for completion of this section is contained within the Quickstart package.

Steps to install Fortress

  1. Download Fortress Quickstart package: iamfortress.org/download

  2. Install Fortress to target machine

    1. Perform Section I - download Fortress QUICKSTART and prepare the package in FORTRESS README-QUICKSTART.html

    2. Edit the build.properties file in root folder of fortress package. Change these settings: 
                              
      
# Unless running everything on localhost, the ldap.host name must match the domain name supplied during Section III - Keys:
ldap.host=fortressdemo2.com
# use 636 for SSL port:
ldap.port=636

# These are needed for client SSL connections with LDAP Server:
ldap.uris=ldap://${ldap.host}:389 ldaps://${ldap.host}:${ldap.port}
enable.ldap.ssl=true
enable.ldap.ssl.debug=false

# Supply the fully qualified pathname to where the truststore resides.  If you are using the demo cert, you may use mytruststore under /src/test/resources/certs:
trust.store=/home/user/fortress/src/test/resources/certs/mytruststore
# the demo trustore password follows:
trust.store.password=changeit
trust.store.set.prop=true

# Supply the fully qualified pathname to where the keystore resides.  If you are using the demo keys, you may use the mykeystore under /src/test/resources/certs:
# the demo keystore password follows:
key.store=/home/user/fortress/src/test/resources/certs/mykeystore
key.store.password=changeit

# These are the fortress demo keys which will be automatically copied to /opt/symas/ssl folder during 'init-slapd' target.
tls.ca.cert.file=ca-cert.pem
tls.cert.file=server-cert.pem
tls.key.file=server-key.pem
    3. Perform Section II - compile Fortress source and build distribution packages in FORTRESS README-QUICKSTART.html.

    4. Perform Section III - install, configure and load OpenLDAP with seed data in FORTRESS README-QUICKSTART.html.

      After completion of this step, Fortress & OpenLDAP will be installed to target machine and using SSL for connections.

    5. Optional - you may test installation by performing Section IV - integration test Fortress and OpenLDAP in the README-QUICKSTART.html guide.

    6. Optional - you may install the RBAC Web admin UI by following Section V - Commander Web Administration demo in the README-QUICKSTART.html guide.

    7. Optional - you may test the RBAC Web admin UI by following Section VI - integration test Commander with Solenium in the README-QUICKSTART.html guide.

    8. Optional - you may connect to the RBAC Web admin UI by following Section VII - connecting to Commander via Web browser in the README-QUICKSTART.html guide.

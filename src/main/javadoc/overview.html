Overview of Fortress Demo # 2 end-to-end Web Application Security

The purpose of this application is to demonstrate principles of Java Web security enforcement. The installation of this application requires completion of the sections listed below.

Sections required for installation:

  1. Install Java 7 (or beyond) SDK
  2. Heartbleed Bug
  3. Managing PKI Keys
  4. Set Hostname Entry
  5. Install Fortress
  6. Install MySQL
  7. Enable Tomcat SSL
  8. Install Fortress Demo #2

Demonstrate end-to-end security enforcement

This sample Web application demonstrates usage of the following security mechanisms:

  1. Java 2 Authentication
  2. Java 2 Coarse-grained Role Authorization
  3. Spring Coarse-grained Page Role Authorization
  4. Medium-grained RBAC Authorization inside Wicket Web Framework
  5. Fine-grained RBAC Authorization within Data Persistence Classes
  6. Java 2 Confidentiality
  7. LDAP v3 Confidentiality
  8. JDBC Confidentiality
  9. Audit Trail of Authentication & Authorization Events

Infrastructure Requirements

Demo Web application prerequisites include:

  1. Java Programming SDK
  2. Fortress RBAC SDK
  3. OpenLDAP Directory Server
  4. MySQL Database Manager
  5. Tomcat Servlet Container
  6. Commander RBAC Web Admin
  7. OpenSSL
  8. JSSE
